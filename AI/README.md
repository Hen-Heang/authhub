# AI Knowledge Base

Personal notes captured while working with Claude on this project. Whenever Claude teaches
something reusable — a pattern, a gotcha, a checklist — write it down here instead of letting
it evaporate at the end of the conversation.

- `Prompts/` — prompts that worked well, worth reusing verbatim or adapting.
- `Claude Commands/` — custom slash commands / workflows built for this repo.
- `Architecture Notes/` — notes on how AuthHub's modules fit together, decisions and tradeoffs.
- `Spring Boot Notes/` — Spring Boot concepts/gotchas learned while building this.
- `PostgreSQL Tips/` — Postgres/JPA/Hibernate tips specific to this project.
- `Security Checklist/` — auth/JWT/MFA security items to check before shipping.
- `Code Review Checklist/` — things to look for when reviewing PRs here.
- `API Design Guide/` — conventions for request/response shapes, versioning, errors.
- `Common Patterns/` — recurring code patterns (mappers, exception handling, etc).

Keep entries short and dated. This is a scratchpad for hard-won knowledge, not formal docs —
formal, current-state docs belong in `docs/`.
